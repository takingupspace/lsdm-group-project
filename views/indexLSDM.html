Introduction

This project is centered around an originally sized dataset of 12.75 GB. The dataset was originally scraped from reddit. The subreddit (internet forum) utilized for this dataset was solely based on Covid-19 related material. The main piece of information contained within this dataset is the "body" field. This field corresponds to an individual user's post in a particular thread on this subreddit.

More

About our dataset

We decided to perform a word count and a sentiment analysis on our dataset. We were able to identify that the most important field in this dataset was the "body" column. We decided to extract this column, and to index each individual row with a unique key, corresponding to a unique individual post. The process that we followed was to first perform extensive data cleaning across the entire dataset. We performed the data cleaning process in multiple steps, simultaneously performing EDA (Exploratory Data Analysis) to make sure that our data made sense. The methodologies used for cleaning our dataset included: converting all text to lower-case, removing brackets, removing punctuation, removing words containing numbers, a few more runs to clean up non-sensical text and additional punctuation, and the last step was to remove stop words. Stop words are words that have little meaning in textual data, such as of, a, and, the, etc.

Word Cloud Visualizations

The first form of analyses that we decided to perform is essentially a word count. The process that we followed was to first, iterate through each row and create a new CSV containing an index and a dictionary of the number of times a word was used for each row. We then performed another iteration to extract the word with the highest count, writing the results to a new CSV with an index and a corresponding row that contained one word and its count. Finally, we utilized the matplot and wordcloud Python Libraries to display a word cloud of our dataset's most used words. As a precursor, our total number of rows in this dataset is around 17 million individual rows. The first word cloud displayed on this page includes the first 2 million rows, the second image shows the word count for the first 9 million rows, and the last image shows all 17 million rows.

Sentiment Analysis Visualizations

We used the same dataset that we cleaned for the word cloud. We went from an originally sized 12.75 GB dataset, to around 6.3 GB. For the sentiment analysis we organized the data by it's original row id and its corresponding body column. Next, we iterated through each row and applied a compound sentiment score and added it to the sentiment score column via Panda Dataframes. Each row also has a sentiment type of "compound" applied to it. We then merge the results intro the original data frame. We then output this data frame to a csv file so that we can generate reports in the future. Lastly, we remove neutral sentiment score and classify each sentiment score as either Positive or Negative.
